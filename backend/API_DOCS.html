HeyBobo Authentication API

REST endpoints for user registration, login, and logout. All responses are JSON.

Base URL https://app.heybobo.ai/api/v1
POST

Register

/auth/register
🔓 Public — no token required

Creates a new user account and returns access + refresh tokens on success.

Request Body (application/json)
FieldTypeRequiredDescription
firstName string required User's first name. Max 50 chars.
lastName string required User's last name. Max 50 chars.
username string required Unique username. 3–30 chars. Letters, numbers, _ and - only.
email string required Valid email address. Must be unique.
password string required Min 8 chars. See password rules below.
role string optional One of student | teacher | creator. Defaults to student.
Password Requirements
  • › Minimum 8 characters
  • › At least one uppercase letter (A–Z)
  • › At least one lowercase letter (a–z)
  • › At least one digit (0–9)
  • › At least one special character: @ $ ! % * ? &
Example Request
// POST https://app.heybobo.ai/api/v1/auth/register
{
  "firstName": "John",
  "lastName":  "Doe",
  "username":  "john_doe",
  "email":     "john@example.com",
  "password":  "SecurePassword@123",
  "role":      "student"
}
Responses
201 Created 
{
  "success": true,
  "data": {
    "accessToken":  "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id":    "64f3c2a1b7e8f100123abc45",
      "email": "john@example.com",
      "name":  "John Doe",
      "role":  "student"
    }
  }
}
400 Bad Request 
{
  "success": false,
  "statusCode": 400,
  "message": "Password must contain uppercase, lowercase, number, and special character"
}
409 Conflict 
{
  "success": false,
  "statusCode": 409,
  "message": "Email already in use"
}

Login

/auth/login
🔓 Public — no token required

Authenticates an existing user with their email/username and password. Returns a short-lived Access Token (JWT) and a Refresh Token.

Request Body (application/json)
FieldTypeRequiredDescription
identifier string required Email address or username.
password string required The user's password.
Tip: The identifier field accepts either an email address (john@example.com) or a username (john_doe).
Example Request
// POST https://app.heybobo.ai/api/v1/auth/login
{
  "identifier": "john@example.com",
  "password":   "SecurePassword@123"
}
Responses
200 OK 
{
  "success": true,
  "data": {
    "accessToken":  "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id":        "64f3c2a1b7e8f100123abc45",
      "email":     "john@example.com",
      "firstName": "John",
      "lastName":  "Doe",
      "username":  "john_doe",
      "role":      "student"
    }
  }
}
401 Unauthorized 
{
  "success": false,
  "statusCode": 401,
  "message": "Invalid credentials"
}
400 Bad Request 
{
  "success": false,
  "statusCode": 400,
  "message": "identifier should not be empty"
}
POST

Logout

/auth/logout
🔐 Protected — Bearer token required

Invalidates the user's refresh token server-side. The access token will expire naturally. Pass the refresh token in the body to revoke it immediately.

Request Headers
HeaderValueRequired
Authorization Bearer <accessToken> required
Content-Type application/json required
Request Body (application/json)
FieldTypeRequiredDescription
refreshToken string optional Pass to revoke this specific refresh token immediately.
Example Request
// POST https://app.heybobo.ai/api/v1/auth/logout
// Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
Responses
200 OK 
{
  "message": "Logged out successfully"
}
401 Unauthorized 
{
  "success": false,
  "statusCode": 401,
  "message": "Unauthorized"
}
POST

Refresh Token

/auth/refresh
🔓 Public — uses refresh token in body

Issues a new access token using a valid refresh token. Call this when the access token expires.

Request Body (application/json)
FieldTypeRequiredDescription
refreshToken string required The refresh token from a previous login or register response.
Example Request
// POST https://app.heybobo.ai/api/v1/auth/refresh
{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
Responses
200 OK 
{
  "success": true,
  "data": {
    "accessToken":  "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
401 Unauthorized 
{
  "success": false,
  "statusCode": 401,
  "message": "Invalid or expired refresh token"
}

Error Codes

HTTP StatusMeaningWhen it occurs
200OKLogin, logout, and refresh succeeded.
201CreatedRegister succeeded — user created.
400Bad RequestValidation failed — missing or malformed fields.
401UnauthorizedWrong credentials, missing/expired token, or invalid refresh token.
403ForbiddenAccount suspended or role not permitted.
409ConflictEmail or username already registered.
429Too Many RequestsRate limit exceeded (max 100 req / 15 min per IP).
500Internal Server ErrorUnexpected server error.

Common Headers

HeaderDescription
Authorization: Bearer <token> Required for all protected endpoints. Use the accessToken returned by login/register.
Content-Type: application/json Required for all POST requests with a body.
